# Bug 修复开发模板

用于定位并修复报错、异常、不生效、交互错误等问题。

## 增强提示词结构

【做什么】：修复[页面/组件名称，如：购物车结算模块、移动端下拉刷新]的[Bug描述，如：结算金额计算错误、下拉刷新触发多次]。

【怎么做】：

1. 定位问题：通过[调试方式，如：console.log打印参数、Chrome DevTools断点调试]定位Bug根因；
2. 修复逻辑：[具体修复方式，如：补充计算逻辑、为事件添加防抖（延迟500ms）]；
3. 代码修改：修改[文件路径/函数名称]并遵循原有代码规范；
4. 测试验证：编写[测试用例，如：连续触发3次仅执行1次]验证修复效果。

【做到何种程度】：

1. Bug修复：目标Bug完全解决，在复现步骤下结果正确；
2. 无副作用：修复后不引入新的Bug，周边功能正常；
3. 代码规范：修改代码符合项目规范，并添加必要注释说明修复原因；
4. 测试覆盖：核心场景测试用例通过，提供测试结果截图/日志。
