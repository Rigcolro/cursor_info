# 性能优化开发模板

用于优化页面、组件或流程中的加载、渲染、滚动、内存等性能问题。

## 增强提示词结构

【做什么】：优化[页面/组件名称，如：首页列表、大数据表格]的[性能问题，如：首屏加载慢、滚动卡顿、内存泄漏]。

【怎么做】：

1. 优化方向：针对[问题原因，如：接口请求过多/渲染节点过多]，采用[优化方案，如：接口合并、懒加载、虚拟列表]；
2. 技术手段：使用[工具/方法，如：Lighthouse分析性能、组件缓存、图片压缩]；
3. 代码调整：[具体调整，如：减少不必要重渲染、优化DOM操作、清理无用事件监听]；
4. 验证方式：通过[性能指标工具，如：Chrome DevTools Performance、FPS计数器]验证优化效果。

【做到何种程度】：

1. 性能指标：[核心指标，如：首屏加载时间从2s降至800ms内、滚动FPS稳定在60帧、内存占用降低30%]；
2. 功能影响：优化后原有功能不受影响，交互逻辑正常，无新的bug产生；
3. 可维护性：优化后的代码保留清晰注释，便于后续维护，不破坏原有代码结构；
4. 验证标准：通过[指定工具，如：Lighthouse]检测，性能评分从[原分数]提升至[目标分数，如：85分以上]。
