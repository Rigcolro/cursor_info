# zl-common-skill

## 适用场景

当用户提出开发需求时，先将原始需求改写为结构化提示词，再执行后续操作。  
优先匹配以下四类任务：

1. 通用功能开发
2. UI 还原开发
3. 性能优化开发
4. Bug 修复开发

---

## 执行流程

1. 识别需求，选择模板。
   - 查阅“模板索引”，根据用户原始需求命中对应的 `文本触发词`，决定要加载的 `templates/*.md` 文件。
   - 若命中多个意图，选择最核心目标作为主模板，其余作为补充约束合并。
   - 若无法判断专项类型，默认加载通用功能开发模板（`templates/general-feature-development.md`），并在增强提示词中标注不确定项。

2. 结合用户提供的信息，根据模板生成“增强提示词（最终版）”。
   - **严格遵守模板**：按需加载已选 `templates/*` 文件，必须严格按照模板的原始结构填充内容。禁止删除核心字段、标题层级或交付标准。禁止用“精简版/摘要版/自定义版”代替模板正文。
   - **填充与合并**：从用户输入中抽取已知信息写入模板。可以根据需求补充字段内容，但不改变约束含义；若多模板命中，以主模板为主体，将补充模板字段作为附加约束合并。
   - **处理缺失信息**：缺失字段必须保留并使用 `[待补充]`，绝不臆造业务事实。若缺少关键执行信息（如设计稿、接口地址、目标文件、复现步骤），暂停并最小化提问补齐。
   - **锁定执行依据**：生成的增强提示词即为最终执行依据，后续的执行计划、编码、调试、验证均必须严格以此为准。

3. 执行规则引用检查。
   - 查阅“规则索引”，采用“双阶段按需匹配”来决定是否加载 `references/*.md` 文件（不默认全量加载）。
   - **阶段A（需求匹配）**：根据用户需求文本匹配 reference 的 `文本触发词`。
   - **阶段B（实现匹配）**：根据后续实现意图匹配 reference 的 `实现触发器`。
   - 命中任一阶段即必须加载对应 reference；若命中但未加载，禁止进入后续步骤。
   - 若代码命中 reference 中的“禁止写法”，必须按规则重写后再输出。

4. 根据“增强提示词”执行编码/调试/验证。
   - 编码、调试、验证必须覆盖增强提示词中的功能、兼容性、性能、体验和交付标准。
   - 若执行中发现增强提示词信息不足，先补充提问或补齐 `[待补充]`，再继续。

5. 输出前执行“反向检测”。
   - 检查生成内容是否出现未加载 template 或 reference 的典型场景。
   - 若出现遗漏，先补加载对应文件并重做增强提示词、实现和校验。
   - 确认无遗漏后，输出最终说明和“规则执行清单”。

---

## 输出规则

- 先输出：`增强提示词（最终版）`
- 再输出：`执行计划（3-6步）`
- 然后进入实际执行（编码/调试/验证/说明）
- 再输出：`规则执行清单`
- 保持术语统一：页面、组件、接口、性能指标、兼容性、交付标准

## 规则索引

### `references/concise-empty-check.md`

- 主题：空字符串与数值 0 的简洁空态判断
- 文本触发词（需求匹配）：空字符串、判空、空态、非空、有效值、0 值、参数校验、string、number、undefined、null
- 实现触发器（实现匹配）：if 条件判断、空字符串判断、数值 0 判断、对象字段存在性判断、默认值兜底、比较前置校验、接口入参校验、undefined 判断、null 判断
- 优先级：高（命中即加载）

### `references/async-await-promise.md`

- 主题：Promise 异步流程统一使用 async/await
- 文本触发词（需求匹配）：Promise、异步、async、await、then、catch、接口请求、异步流程
- 实现触发器（实现匹配）：`.then(`、`.catch(`、Promise 链式调用、异步接口调用、异步错误处理
- 优先级：高（命中即加载）

## 使用示例

用户输入（原始）：  
“移动端个人中心页样式乱了，顺便把加载速度提一下。”
处理方式：

1. 命中“样式乱了”，加载 `templates/ui-restoration-development.md` 作为主模板。
2. 命中“加载速度”，加载 `templates/performance-optimization-development.md` 作为补充模板。
3. 将两个模板中的字段合并为一份 `增强提示词（最终版）`，缺失信息保留 `[待补充]`。
4. 执行规则引用检查：若实现中涉及判空逻辑，按需加载 `references/concise-empty-check.md`。
5. 用增强提示词驱动后续实现与验证，并输出包含模板和 references 的“规则执行清单”。
